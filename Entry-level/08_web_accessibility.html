1. 웹 접근성 기본 원칙 (WCAG 2.1)

웹 접근성은 4가지 기본 원칙을 따릅니다: 인식 가능성, 운용 가능성, 이해 가능성, 견고성

인식 가능성 (Perceivable)

ℹ️
모든 사용자가 정보를 인식할 수 있어야 합니다
  • 이미지에 대체 텍스트 제공
  • 충분한 색상 대비 확보
  • 동영상에 자막 제공
  • 텍스트 크기 조절 가능

색상 대비 예제

좋은 대비 (21:1) - 검은 텍스트 on 흰 배경 ✅ WCAG AAA 통과
보통 대비 (3.5:1) - 회색 텍스트 on 연한 회색 배경 ⚠️ WCAG AA 통과
나쁜 대비 (1.4:1) - 흰 텍스트 on 노란 배경 ❌ WCAG 실패

2. 시맨틱 HTML과 ARIA

의미 있는 HTML 구조와 ARIA 속성으로 스크린 리더 호환성을 높입니다.

시맨틱 HTML 예제

<!-- 좋은 예시 -->
<article>
  <header>
    <h2>안건의 의료IT 프로젝트</h2>
    <time datetime="2024-01-15">2024년 1월 15일</time>
  </header>
  <section>
    <p>프로젝트 내용...</p>
  </section>
  <footer>
    <p>작성자: 유진우 (의료IT)</p>
  </footer>
</article>

ARIA 속성 활용

⚠️
ARIA 사용 시 주의사항
  • 시맨틱 HTML을 우선 사용
  • ARIA는 보완적으로 활용
  • 올바른 역할(role)과 속성 사용
