1. CSS 설계 방법론 비교

BEM (Block Element Modifier)

컴포넌트 기반의 명명 규칙으로, 블록-요소-수정자 구조를 사용합니다.

  • 명확한 네이밍 컨벤션
  • 스타일 충돌 방지
  • 재사용 가능한 컴포넌트
  • 팀 협업에 유리
/* Block */ .card { } /* Element */ .card__header { } .card__title { } .card__body { } /* Modifier */ .card--primary { } .card--elevated { }

OOCSS (Object Oriented CSS)

구조와 스킨을 분리하여 재사용 가능한 CSS 객체를 만드는 방법론입니다.

  • 구조와 스킨 분리
  • 컨테이너와 콘텐츠 분리
  • 높은 재사용성
  • 성능 최적화
/* Structure */ .btn { padding: 12px 24px; } /* Skin */ .btn--primary { background: blue; } .btn--secondary { background: gray; } /* Size */ .btn--large { padding: 16px 32px; }

SMACSS (Scalable and Modular CSS)

CSS를 5가지 카테고리로 분류하여 관리하는 확장 가능한 아키텍처입니다.

  • Base, Layout, Module, State, Theme
  • 체계적인 구조화
  • 확장성 보장
  • 명확한 역할 분담
/* Base */ body, h1, p { } /* Layout */ .l-header, .l-main { } /* Module */ .nav, .card { } /* State */ .is-active, .is-hidden { }

Atomic CSS

작은 단위의 유틸리티 클래스를 조합하여 스타일을 구성하는 방법론입니다.

  • 원자 단위 클래스
  • 빠른 개발 속도
  • 일관된 디자인 시스템
  • 번들 크기 최적화
/* Utility Classes */ .m-4 { margin: 1rem; } .p-2 { padding: 0.5rem; } .text-center { text-align: center; } .bg-primary { background: #007bff; }

3. OOCSS 구조와 스킨 분리

⚠️
OOCSS 핵심 원칙

구조(Structure)와 스킨(Skin)을 분리하여 재사용성을 높이고, 컨테이너와 콘텐츠를 독립적으로 관리합니다.

버튼 컴포넌트 예시

/* 구조 정의 */ .btn { display: inline-flex; align-items: center; padding: var(--spacing-sm) var(--spacing-md); border: 2px solid transparent; border-radius: var(--border-radius-md); font-weight: var(--font-weight-medium); cursor: pointer; transition: all var(--transition-fast); } /* 스킨 정의 */ .btn--primary { background-color: var(--color-primary-500); color: white; } .btn--secondary { background-color: var(--color-secondary-500); color: white; } /* 크기 변형 */ .btn--sm { padding: var(--spacing-xs) var(--spacing-sm); } .btn--lg { padding: var(--spacing-md) var(--spacing-xl); }
